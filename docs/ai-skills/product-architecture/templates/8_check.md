# Layer 8: CHECK - 最终上线与写代码前检查

> [!IMPORTANT]
> **在动笔编写第一行代码前，必须由 PM 和架构师共同闭眼回答以下问题。若其中任何一项回答是“不知道”或模糊不清的，必须立即暂停开发，退回前面的步骤重新梳理！**

## 1. 为什么存在？ (Why does it exist?)
* 用一句话概括：这个产品在宇宙中存在的独特价值是什么？

## 2. 解决什么问题？ (What problem does it solve?)
* 具体的、可衡量的用户痛点是什么？谁会因为这个问题得不到解决而痛苦？

## 3. 核心机制是什么？ (What is the core mechanism?)
* 用一句话说清底层的决策闭环或信息流（输入 → 行动 → 反馈）。

## 4. MVP是什么？ (What is the MVP?)
* 砍掉了 80% 想法后，最终确认只开发的最小闭环功能是什么？

## 5. 数据如何积累？ (How does data accumulate?)
* 随着用户使用，系统沉淀了哪些能反映成败因果的行为数据？

## 6. 护城河是什么？ (What is the moat?)
* 如果底层大模型在一夜之间发生变动，为什么用户依然不会离开我们？

---

**最终结论：**
*  [ ] **通过自检，允许开始编写代码！**
*  [ ] **未通过自检，拒绝开发，退回重修。**
