# Layer 4: MVP - 最小可行性产品

## 1. 核心价值主张 (Core Value Proposition)
* **一句话说清**：这个 MVP 到底能给用户解决哪个最关键的痛点？

## 2. 7天极限压缩设计 (The 7-Day Cutoff)
* 如果只有 7 天的开发时间，必须无条件删除哪些功能？
* 必须留下哪些功能以确保它依然可用？

## 3. 功能对比列表 (Feature Inclusion & Exclusion)

| 核心功能 (Must Have) | 次要/砍掉功能 (Nice to Have) | 砍掉原因 (Exclusion Reason) |
| :--- | :--- | :--- |
| **例**：图片上传与苏格拉底对话 | 社交分享与朋友圈对比海报 | 属于运营级功能，不影响核心学习闭环 |
| | | |
| | | |

## 4. MVP 验证成功指标 (Success Criteria)
* 用户完成多少个完整闭环才算需求得到验证？
