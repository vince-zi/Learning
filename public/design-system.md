# Learniny 设计规范 v1

> 目的：任何 AI（Claude / Cursor / Windsurf / DeepSeek）在写 Learniny 的前端代码前，
> 先读这份文档，确保跨 IDE、跨 session 的视觉输出收敛到同一个方向。
> 触发条件：涉及任何前端页面、组件、样式的编写或修改。

---

## 0. 产品定位（决定所有视觉判断的锚点）

Learniny 不是"英语学习软件"的调性，是"用于练习英语直觉的思考工具"。

- 参照系：Notion / Linear / Arc，而不是多邻国 / 流利说
- 用户画像：创业者、职场人士，要效率感和掌控感，不要被哄着学习
- 核心隐喻：混沌中的知识粒子 → 结晶成直觉。这个隐喻**只用于非核心页面**
  （星图、成就、首页），对话等核心操作页面必须克制
- 复杂度分层原则：后台算法（ZPD、认知负荷、多层纠错策略）永远隐形，
  系统越复杂，界面必须越简单

## 2. 字体

```
@import url('https://fonts.googleapis.com/css2?family=Fraunces:opsz,wght@9..144,400;9..144,500;9..144,600&family=Inter:wght@400;500;600&family=JetBrains+Mono:wght@400;500&display=swap');
```

| 角色 | 字体 | 用途 |
|---|---|---|
| Display | `'Fraunces', serif` | 页面标题、节点名、弹窗大标题——**只用于标题，不用于正文** |
| Body | `Inter, sans-serif` | 对话内容、按钮、正文 |
| Mono/Data | `'JetBrains Mono', monospace` | CEFR标签、分类标签、时间戳、统计数字标签——所有"系统数据感"的小字 |

原则：serif 传达"这是一件值得认真对待的事"，mono 传达"这是系统在报告数据"，
两者都克制使用，正文永远是 Inter。

## 3. 组件语言

### 3.1 卡片 — 两种圆角逻辑
- **常规卡片/气泡**：标准圆角 `border-radius: 10-16px`，AI气泡用不对称圆角
  `4px 16px 16px 16px`（左上角尖，模拟"从头像发出"），用户气泡镜像。
- **底部弹窗（Sheet）**：顶边用切角，不用普通圆角。这是产品的**签名细节**，
  呼应"结晶"隐喻，专用于一切从底部滑出的浮层（diff卡、节点详情、温习流程）：
  ```css
  clip-path: polygon(0% 14px, 6% 0%, 94% 0%, 100% 14px, 100% 100%, 0% 100%);
  ```
  不要在常规卡片上用切角，否则失去"这是一个特殊时刻"的信号意义。

### 3.2 状态标签（Tag）
统一样式：`JetBrains Mono, 10-11px`，`padding: 2px 7px`，`border-radius: 5px`，
背景用语义色的 `22`(浅) 或 `33`(略深) 透明度叠加，文字用实色。
例：CEFR等级标 `background: goldSoft; color: gold`。

### 3.3 按钮
- 主按钮：实心金色背景 `#C9A15D`，文字深色 `#181410`，`border-radius: 10px`，
  `font-weight: 600`。这是唯一允许纯色实心的场景（对应"结晶完成/确认"的动作）。
- 次按钮：透明背景 + `1px solid divider` 描边，文字用 `textPrimary`。
- 不用渐变按钮，不用阴影堆叠。

### 3.4 纠错三态视觉规则（核心交互，务必保持一致）
1. **Recast（不打断）**：无任何视觉标记，正确说法自然嵌入 AI 下一句普通气泡里。
2. **Metalinguistic hint（元语言提示）**：AI气泡上方加一个 `tealSoft` 背景的小标签条
   （图标+文字），气泡本身背景换成 `surfaceAIHint`，边框 `teal 33%`。不要用弹窗，
   不要打断对话流。
3. **Diff 卡（用户主动点开才看到）**：用户消息右下角一个 8-9px 的金色小圆点作为
   唯一入口提示，默认不展开。点击后走 3.1 的切角 Sheet，原句用 `coral` 划线，
   修正后用 `gold` 高亮+发光（`box-shadow: 0 0 8px goldSoft`），中间用一个
   sparkle 图标分隔，不做逐字 diff 的机械对比，只标出真正变化的片段。

### 3.5 星图节点状态
- 已掌握：实心 `gold`，外圈叠加低透明度光晕
- 进行中/当前：实心，外圈脉冲动画光环（呼吸感，2.4s循环）
- 锁定：仅描边、无填充，配一个小锁图标，整体 `opacity: 0.55`
- 连线：已解锁路径用金色实线，未解锁用灰色虚线

## 4. 动效原则

- 只在"结晶时刻"给强反馈：节点解锁、错题消灭、CEFR升级——这些才配得上动画。
- 弹窗统一 `slideUp 0.28s ease-out`，不用更花哨的入场。
- 对话流本身零动效干扰，用户在思考时不该被任何动态元素分心。
- 脉冲/呼吸类动画只用于"当前进行中"的单一焦点，同一屏最多一个元素在动。

## 5. 信息架构（不要做仪表盘式平铺）

底部导航固定四个入口，不展示六个模块的技术分类：

```
对话（默认页，占用户90%时间） · 星图 · 错题本 · 我的
```

- 学习理论（ZPD/认知负荷/纠错策略）永远是后台引擎，用户无感知
- 星图不是日常必经页，定位为"周回顾"场景，不要求高频打开
- 3D 版星图作为"探索模式"可选入口，默认给 2D/SVG 版本（性能和可控性更好）

---

## 6. 反模式（明确不要做的事）

- ❌ 不用近黑背景+荧光绿/朱红的"AI感"配色（太模板化）
- ❌ 不用多邻国式的卡通吉祥物、经验条、连续打卡火苗弹窗
- ❌ 不在对话界面里用花哨卡片打断 iMessage 式气泡流
- ❌ 不把"系统检测到你情绪低落"做成一条提示bar——情感感知应该只体现在
  AI语气变化上，不该有"被监视感"的系统提示
- ❌ 星图不要默认用 3D/Three.js 全屏（手机端性能和发现性都差，参考本产品
  MindWeave 项目里的着色器技术积累留到"探索模式"这个可选场景里复用）
- ❌ 切角 Sheet 样式不要滥用到所有卡片上，只用于底部浮层
